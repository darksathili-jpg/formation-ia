Complexité algorithmiqueMesurer l'efficacité avec la notation Grand O
La complexité temporelle indique le nombre d'opérations effectuées en fonction de la taille $n$ de l'entrée. La notation Grand O exprime le comportement dans le pire cas.
Notations courantes $O(1)$  — constant (indépendant de $n$)
$O(n)$  — linéaire (une boucle sur $n$ éléments)
$O(n^2)$ — quadratique (deux boucles imbriquées)

Parcours de tableaux — Patterns clésLes techniques essentielles en Python
La plupart des exercices reposent sur un parcours séquentiel avec accumulation. Voici les patterns fondamentaux :
PYTHON # Parcours par indice (pour comparer tab[i] et tab[i+1]) for i in range(len(tab) - 1): # comparer tab[i] et tab[i+1] # Indice ET valeur simultanément for i, val in enumerate(tab): # i = indice, val = tab[i] # Parcours à rebours (fin → début) for i in range(len(tab) - 1, -1, -1):
Court-circuit : retourner False dès qu'une condition est violée interrompt immédiatement la boucle — c'est l'optimisation classique des algorithmes de vérification.

Retour anticipé : return True ou return False en cours de boucle interrompt immédiatement la fonction — souvent plus lisible qu'un indicateur booléen.

Exercices 1, 2, 3 — Parcours et accumulation

Cours — Vérification de propriétéParcours de paires avec condition d'arrêt
Pour vérifier qu'un tableau est trié, on parcourt les paires d'éléments consécutifs. Dès qu'une paire viole la condition, on retourne False — sans finir la boucle.
Condition de tri croissant $\forall\, i \in [0,\, n-2],\quad \text{tab}[i] \leq \text{tab}[i+1]$
ALGO Algorithme verifie(tab) : Pour i de 0 à len(tab)−2 : Si tab[i] > tab[i+1] : Retourner False ← court-circuit Retourner True
Complexité : $O(n)$ au pire (tout parcourir), $O(1)$ au mieux (violation dès la 1ère paire). Tableau à 1 élément : boucle vide → True.
💡 Indices
  • Utiliser range(len(tab) - 1) pour ne pas dépasser
  • Égalités autorisées : condition stricte >, pas >=

Cours — Recherche du maximum avec tous ses indicesAccumulation en une passe + reset de la liste
Chercher le maximum et tous ses indices se fait en une seule passe $O(n)$. La difficulté : quand on trouve un nouveau maximum, il faut réinitialiser la liste des indices.
ALGO maxi ← tab[0]  ; indices ← [0] Pour i de 1 à len(tab)−1 : Si tab[i] > maxi : maxi ← tab[i] ; indices ← [i] ← reset ! Sinon si tab[i] == maxi : indices.append(i) Retourner (maxi, indices)
Complexité : $O(n)$ — une seule boucle. Espace $O(k)$ avec $k$ = nombre d'occurrences du maximum.

Le reset de la liste d'indices est le point délicat : un nouveau maximum rend obsolètes tous les indices précédents.

Cours — Moyenne pondérée et cas limiteDouble accumulateur, None, tuples
Une moyenne pondérée attribue un poids différent à chaque valeur. Chaque entrée est un tuple (valeur, coefficient).
Formule mathématique $$\bar{x} = \frac{\displaystyle\sum_{i} v_i \cdot c_i}{\displaystyle\sum_{i} c_i}$$ $v_i$ = valeur  ·  $c_i$ = coefficient
Cas particulier : si $\sum c_i = 0$, la division serait par zéro → retourner None (absence de valeur en Python).
💡 Indices
  • Déstructurer le tuple : for valeur, coeff in tab
  • Accumuler somme_ponderee et somme_coeffs
  • Tester somme_coeffs == 0 avant de diviser
  • Le / donne directement un float
Complexité : $O(n)$ — un seul parcours de la liste de tuples.

Exercices 4, 5 — Booléens et aléatoire

Cours — Exploiter le tri pour détecter les doublonsVoisins adjacents → $O(n)$ au lieu de $O(n^2)$
Dans une liste déjà triée, deux valeurs identiques sont forcément adjacentes. Il suffit de comparer chaque élément avec son voisin suivant — pas besoin de double boucle !
Propriété clé Liste triée + doublon $\Rightarrow\, \exists\, i$ tel que $\text{tab}[i] = \text{tab}[i+1]$

Avec tri : $O(n)$  ·  Sans tri (naïf) : $O(n^2)$
Avantage du tri : exploiter la structure de l'entrée réduit la complexité. C'est un principe général de l'algorithmique.

Liste vide ou à un seul élément ne peut pas contenir de doublons — à traiter en premier pour éviter une erreur d'index.

Exercices 6, 7 — Recherche et fusion

Cours — Recherche de la dernière occurrenceParcours inverse et valeur sentinelle
Pour trouver la dernière occurrence d'un élément, on parcourt le tableau à rebours et on retourne l'indice dès la première valeur trouvée — qui est la dernière dans l'ordre original.
ALGO # Parcours décroissant Pour i de len(tab)−1 jusqu'à 0 : Si tab[i] == n : Retourner i Retourner len(tab) ← valeur sentinelle
Valeur sentinelle : retourner len(tab) quand l'élément est absent est une valeur hors domaine qui signale "non trouvé" sans lever d'exception.
Complexité : $O(n)$ dans le pire cas. $O(1)$ dans le meilleur cas (dernier élément = valeur cherchée).

Cours — Fusion de listes triéesDeux pointeurs — cœur du tri fusion $O(n \log n)$
Fusionner deux listes triées est la brique du tri fusion. On utilise deux pointeurs i et j, et on avance celui qui pointe vers la plus petite valeur.
tab1
[3, 5]
+
tab2
[2, 5]
→
résultat
[2, 3, 5, 5]
ALGO i, j ← 0, 0 ; result ← [] TantQue i < len(tab1) et j < len(tab2) : Si tab1[i] ≤ tab2[j] : ajouter tab1[i] ; i++ Sinon : ajouter tab2[j] ; j++ result += tab1[i:] + tab2[j:] ← restes
Complexité : $O(n + m)$ — optimal. Le tri fusion complet atteint $O(n \log n)$ en appliquant cette fusion récursivement.

Cours — Représentation binairePuissances de 2 · Algorithme de Horner
En base 2, chaque bit à la position $i$ depuis la droite vaut $2^i$. Le tableau va du bit de poids fort (gauche, $2^{n-1}$) au bit de poids faible (droite, $2^0$).
Formule de conversion $$\text{décimal} = \sum_{i=0}^{n-1} \text{tab}[i] \cdot 2^{\,n-1-i}$$ Exemple : [1,0,1,0,0,1,1] → $n=7$
$1{\cdot}64 + 0{\cdot}32 + 1{\cdot}16 + 0 + 0 + 1{\cdot}2 + 1{\cdot}1 = \mathbf{83}$

Méthode de Horner : sans calculer de puissance : accumuler $r \leftarrow r \times 2 + \text{tab}[i]$ à chaque étape. Essaie de l'implémenter !

Complexité : $O(n)$ — une seule passe sur les $n$ bits du tableau.
