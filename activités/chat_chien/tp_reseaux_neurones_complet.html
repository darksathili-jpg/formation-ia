2.A — Les fonctions d'activation ReLU et Sigmoïde

Les fonctions d'activation introduisent de la non-linéarité dans le réseau. Sans elles, empiler des couches serait inutile (tout resterait linéaire).

ReLU — couches cachées ReLU(z) = max(0, z)
→ Simple, efficace : éteint les neurones négatifs (z < 0 → sortie = 0)
Sigmoïde — couche de sortie σ(z) = 1 / (1 + e⁻ᶻ)
→ Compresse dans [0 ; 1] : interprétable comme une probabilité
Dérivées (pour la rétropropagation) ReLU'(z) = 1 si z > 0, sinon 0
σ'(z) = σ(z) · (1 − σ(z))

2.B — La fonction de perte : Binary Cross-Entropy Mesurer l'erreur de classification

L'entropie croisée binaire (BCE) mesure à quel point la prédiction ŷ s'éloigne de la réalité y. Plus ŷ est proche de y, plus la perte est faible.

Binary Cross-Entropy L(y, ŷ) = −[y · log(ŷ) + (1−y) · log(1−ŷ)]
Analogie : Imagine que tu estimes à 90% que c'est un chien (ŷ=0.9), mais c'était un chat (y=0). Ta pénalité = −log(0.1) ≈ 2.3 : très élevée ! Si tu avais dit 51% chien, ta pénalité = −log(0.49) ≈ 0.7 : bien plus faible.
Gradient de la perte par rapport à ŷ ∂L/∂ŷ = −y/ŷ + (1−y)/(1−ŷ)
→ C'est le point de départ de la rétropropagation

# ══════════════════════════════════════════════════════════════════

class ReseauChatsChiens(nn.Module):
    """
    Réseau de neurones PyTorch : 784 → 64 → 64 → 1.
    
    Hérite de nn.Module : PyTorch gère automatiquement les paramètres
    et le calcul des gradients.
    """
    
    def __init__(self):
        super().__init__()
        
        # Définition des couches linéaires (équivalent à nos "Couche")
        # nn.Linear(in, out) = couche avec poids et biais, sans activation
        self.couche1 = nn.Linear(784, 64)  # 784 → 64
        self.couche2 = nn.Linear(64, 64)   # 64 → 64
        self.couche3 = nn.Linear(64, 1)    # 64 → 1
        
        # Initialisation He (comme en Phase 2)
        nn.init.kaiming_normal_(self.couche1.weight, mode='fan_in', nonlinearity='relu')
        nn.init.kaiming_normal_(self.couche2.weight, mode='fan_in', nonlinearity='relu')
        nn.init.xavier_normal_(self.couche3.weight)
    
    def forward(self, x):
        """
        Propagation avant : équivalent à notre méthode feedforward().
        PyTorch appelle cette méthode automatiquement.
        
        Paramètre :
            x (Tensor) : batch d'images, shape (batch_size, 784)
        
        Retour :
            Tensor : prédictions, shape (batch_size, 1)
        """
        # Couche 1 : linéaire + ReLU
        x = torch.relu(self.couche1(x))
        
        # Couche 2 : linéaire + ReLU
        x = torch.relu(self.couche2(x))
        
        # Couche 3 : linéaire + Sigmoïde
        x = torch.sigmoid(self.couche3(x))
        
        return x


# Instanciation et affichage

        # 5) Mise à jour des poids

    modele_pt.eval()  # Mode évaluation (désactive dropout)

Phase 4 — CNN vs MLP : Pourquoi le CNN écrase le MLP sur les images ?

Dans les phases précédentes, notre MLP aplatissait chaque image en un vecteur de 784 valeurs. Ce faisant, il détruisait toute notion de voisinage spatial : le pixel en haut à gauche et son voisin immédiat devenaient deux informations sans lien. Le CNN (Réseau de Neurones Convolutif) résout ce problème en raisonnant directement sur des régions locales de l'image.

MLP 784→64→64→1
  • Aplatit l'image → perd la structure 2D
  • Un poids différent par pixel et par position
  • Un chat à gauche ≠ un chat à droite pour lui
  • ~55 000 paramètres, overfitting rapide
  • Ne peut pas généraliser la position
  • Précision typique : 60–68 %
VS
CNN conv→pool→dense→1
  • Travaille sur des patches 3×3 ou 5×5
  • Partage les poids → 9 paramètres balayent toute l'image
  • Invariant par translation (chat = chat partout)
  • ~121 000 paramètres mais meilleure généralisation
  • Hiérarchie de features : bords → formes → objets
  • Précision typique : 78–86 %

Étape 1 — Comprendre la convolution

Une convolution applique un petit filtre (kernel) en le faisant glisser sur toute l'image. À chaque position, on calcule le produit scalaire entre le filtre et le patch d'image. Le résultat est une feature map qui révèle où le motif est présent.

Opération de convolution S(i, j) = Σm Σn I(i+m, j+n) · K(m, n)
Économie de paramètres Filtre 3×3 = 9 poids pour toute l'image
MLP équivalent = 784 × 64 = 50 176 poids
Analogie : un filtre CNN, c'est comme une loupe qu'on fait glisser sur l'image. Elle cherche toujours le même motif (bord, courbe, texture) peu importe sa position.

        x = self.drop(self.relu(self.fc1(x)))    # → (N, 64)

Activation function Architecture
→ Fonction d'activation
Fonction non-linéaire appliquée à la sortie d'un neurone. Sans elle, empiler des couches ne servirait à rien (tout resterait une seule transformation linéaire). Exemples : ReLU, Sigmoïde, Tanh, Softmax.
ReLU Architecture
→ Unité Linéaire Rectifiée
Fonction d'activation la plus utilisée dans les couches cachées : f(x) = max(0, x). Simple, rapide à calculer, évite le problème de "vanishing gradient" qui affecte la Sigmoïde en profondeur.
Sigmoid / Sigmoïde Architecture
→ Sigmoïde
Fonction d'activation qui compresse toute valeur entre 0 et 1 : σ(x) = 1/(1+e⁻ˣ). Utilisée en couche de sortie pour la classification binaire. Interprétable comme une probabilité.
Softmax Architecture
→ Softmax
Généralisation de la Sigmoïde pour les classifications à plus de 2 classes. Convertit un vecteur de scores en probabilités dont la somme vaut 1. Ex. : chien=0.85, chat=0.12, lapin=0.03.
Tanh Architecture
→ Tangente hyperbolique
Fonction d'activation qui compresse les valeurs entre −1 et +1. Centrée en zéro contrairement à la Sigmoïde, ce qui accélère parfois la convergence. Encore utilisée dans les RNN/LSTM.
Dropout Architecture
→ Abandon (rarement traduit)
Technique de régularisation qui désactive aléatoirement un pourcentage de neurones à chaque batch d'entraînement. Réduit l'overfitting en empêchant le réseau de trop dépendre de certains chemins.
Batch Normalization Architecture
→ Normalisation par batch
Technique qui normalise les activations à l'intérieur du réseau après chaque couche. Accélère l'entraînement, stabilise le gradient, permet des learning rates plus élevés. Très courant dans les CNN modernes.
Flatten Architecture
→ Aplatissement
Opération qui transforme un tenseur multidimensionnel en vecteur 1D. Dans un CNN, on fait Flatten pour passer des feature maps 2D au classifieur Dense final (ex. 5×5×64 → 1600).
Embedding Architecture
→ Plongement vectoriel / Représentation
Représentation dense d'une donnée (mot, image, utilisateur…) sous forme d'un vecteur de nombres réels. Permet au réseau de travailler sur des données symboliques. Fondamental en NLP (Word2Vec, BERT).
Residual connection / Skip connection Architecture
→ Connexion résiduelle / Raccourci
Connexion qui court-circuite une ou plusieurs couches en ajoutant l'entrée à la sortie : y = F(x) + x. Inventé dans ResNet (2015), permet d'entraîner des réseaux très profonds sans dégradation du gradient.
Attention mechanism Architecture
→ Mécanisme d'attention
Mécanisme qui permet au réseau de pondérer dynamiquement l'importance de différentes parties de l'entrée. Base des Transformers (GPT, BERT). Chaque token "fait attention" aux autres selon leur pertinence.
Convolution / Conv2D CNN
→ Convolution / Couche de convolution
Opération qui applique un filtre (kernel) en le faisant glisser sur toute l'image. À chaque position, produit scalaire entre le filtre et le patch. Détecte des motifs locaux (bords, textures, formes).
Kernel / Filter CNN
→ Noyau / Filtre
Petite matrice de poids (souvent 3×3) représentant un motif à détecter. Un filtre 3×3 = 9 paramètres appliqués à toute l'image. Vs MLP : 784×64 = 50 176 paramètres pour la même couche.
Feature map CNN
→ Carte d'activation
Image résultante après application d'un filtre de convolution. Chaque pixel de la feature map indique l'intensité de présence du motif recherché à cette position de l'image originale.
Pooling / MaxPooling CNN
→ Sous-échantillonnage
Réduit la taille d'une feature map en ne gardant que la valeur maximale dans chaque fenêtre 2×2. Divise chaque dimension par 2, apporte une invariance aux petits décalages, réduit l'overfitting.
Weight sharing CNN
→ Partage des poids
Principe clé du CNN : le même filtre (les mêmes 9 poids) est appliqué à toutes les positions de l'image. Détecte le même motif partout. Rend le réseau invariant à la translation.
Stride / Padding CNN
→ Pas de déplacement / Rembourrage
Stride : pas de déplacement du filtre (1 = pixel par pixel, 2 = saute un pixel). Padding='same' : ajout de zéros sur les bords pour conserver les dimensions de sortie.
Receptive field CNN
→ Champ réceptif
Région de l'image originale qu'un neurone "voit" en pratique. Après plusieurs couches de convolution + pooling, un neurone profond a un champ réceptif large et peut détecter des structures complexes à grande échelle.
Global Average Pooling CNN
→ Moyenne globale par canal
Alternative au Flatten qui calcule la moyenne de chaque feature map entière. Réduit drastiquement les paramètres de la couche Dense finale. Utilisé dans les architectures modernes (MobileNet, EfficientNet).
Transfer learning CNN
→ Apprentissage par transfert
Réutilisation d'un réseau pré-entraîné (ex. VGG16 sur ImageNet) comme point de départ pour une nouvelle tâche. Les premières couches (détecteurs de bords, textures) sont communes à presque toutes les images.
Data augmentation CNN
→ Augmentation de données
Technique qui génère artificiellement de nouvelles images d'entraînement en appliquant des transformations : rotation, flip, zoom, décalage, bruit. Augmente la diversité du dataset sans collecter plus de données.
Epoch Entraînement
→ Époque
Un passage complet sur l'intégralité du dataset d'entraînement. 20 epochs = chaque image vue 20 fois. Trop peu → underfitting. Trop → overfitting. La courbe de validation indique quand s'arrêter.
Batch / Mini-batch Entraînement
→ Lot
Petit groupe d'images traité ensemble avant une mise à jour des poids. batch_size=32 → 625 mises à jour par epoch (20 000/32). Petit batch = gradient bruité mais meilleure généralisation.
Learning rate Entraînement
→ Taux d'apprentissage (noté α)
Taille du pas lors de la descente de gradient : w ← w − α · ∂L/∂w. Trop grand → oscillations ou divergence. Trop petit → convergence très lente. Adam l'ajuste automatiquement.
Overfitting Entraînement
→ Surapprentissage
Le réseau mémorise les données d'entraînement au lieu de généraliser. Symptôme : précision train élevée, précision test faible. Remèdes : Dropout, régularisation L2, plus de données, data augmentation, early stopping.
Underfitting Entraînement
→ Sous-apprentissage
Le modèle n'est pas assez puissant pour capturer les patterns du dataset. Précision médiocre en train et en test. Remèdes : plus de couches, plus de neurones, plus d'epochs, modèle plus complexe.
Early stopping Entraînement
→ Arrêt précoce
Technique qui interrompt l'entraînement quand la perte de validation cesse de s'améliorer (plateau). Évite l'overfitting sans devoir fixer à l'avance le nombre d'epochs optimal.
Train / Validation / Test split Entraînement
→ Découpage train / validation / test
Train : données pour apprendre les poids. Validation : données pour ajuster les hyperparamètres. Test : données jamais vues, pour la mesure de performance finale. Dans notre TP : 80% / 10% / 10%.
Cross-validation / K-fold Entraînement
→ Validation croisée
Technique d'évaluation qui divise le dataset en K parties, entraîne K fois en utilisant tour à tour chaque partie comme jeu de test. Donne une estimation plus robuste de la performance réelle du modèle.
Regularization L1 / L2 Entraînement
→ Régularisation
Ajout d'une pénalité sur les poids dans la fonction de perte pour éviter l'overfitting. L2 (Ridge) : pénalise les poids élevés. L1 (Lasso) : force certains poids exactement à zéro (sélection de features).
Convergence Entraînement
→ Convergence
État atteint quand la perte ne diminue plus significativement d'une epoch à l'autre. Indique que le modèle a trouvé un minimum (local ou global) de la fonction de perte. Visible sur les courbes d'apprentissage.
Backpropagation Mathématiques
→ Rétropropagation du gradient
Algorithme central du deep learning. Calcule comment chaque poids contribue à l'erreur en propageant les gradients de la sortie vers l'entrée (règle de la chaîne des dérivées). Codé manuellement en Phase 2.
Gradient descent Mathématiques
→ Descente de gradient
Méthode d'optimisation qui met à jour les poids dans la direction opposée au gradient de la perte. Analogie : descendre une montagne en cherchant toujours la pente la plus raide. Trois variantes : batch, stochastique, mini-batch.
Loss function Mathématiques
→ Fonction de perte
Mesure l'écart entre la prédiction du réseau et la vraie réponse. BCE pour la classification binaire, Cross-entropy pour le multi-classe, MSE pour la régression. Le réseau minimise cette valeur.
Adam optimizer Mathématiques
→ Optimiseur Adam
Adaptive Moment Estimation. Combine momentum (accélération dans la bonne direction) et RMSProp (learning rate adaptatif par paramètre). Par défaut dans la plupart des architectures modernes. lr=0.001 est un bon point de départ.
Hyperparameter Mathématiques
→ Hyperparamètre
Paramètre de configuration non appris par le réseau. Exemples : learning rate, batch_size, nombre de couches, taille des couches, taux de dropout. Réglés par le concepteur, souvent par essai/erreur ou grid search.
Accuracy Mathématiques
→ Précision / Taux de réussite
Proportion de prédictions correctes : bonnes prédictions / total. 82% = 82 images correctement classifiées sur 100. Attention : trompeuse sur des datasets déséquilibrés (99% de chats → accuracy de 99% en prédisant toujours "chat").
Precision / Recall / F1-score Mathématiques
→ Précision / Rappel / Score F1
Precision : parmi les prédictions "chien", combien sont vraiment des chiens. Recall : parmi tous les vrais chiens, combien a-t-on trouvé. F1 : moyenne harmonique des deux. Plus robuste que l'accuracy seul.
Vanishing / Exploding gradient Mathématiques
→ Disparition / Explosion du gradient
Problèmes dans les réseaux profonds : vanishing = les gradients deviennent quasi-nuls dans les premières couches (la Sigmoïde en profondeur). Exploding = les gradients divergent. ReLU et les connexions résiduelles atténuent ces phénomènes.
Weight initialization Mathématiques
→ Initialisation des poids
Valeurs initiales des poids avant l'entraînement. Cruciale : tous à zéro → symétrie, rien n'apprend. He initialization (√2/n) est optimale pour ReLU. Xavier/Glorot pour Tanh et Sigmoïde.
Tensor Mathématiques
→ Tenseur
Généralisation des matrices à n dimensions. Un vecteur = tenseur 1D, une image = tenseur 3D (H×W×C), un batch d'images = tenseur 4D (N×H×W×C). Unité de base de NumPy, PyTorch et TensorFlow.
Dataset Données
→ Jeu de données
Ensemble de données d'entraînement. Dans notre TP : 24 998 images (X) avec leurs étiquettes (Y : 0=chat, 1=chien). La qualité et la quantité du dataset sont souvent plus importantes que l'architecture du réseau.
Label / Ground truth Données
→ Étiquette / Vérité terrain
La réponse correcte associée à chaque exemple d'entraînement. Dans notre TP : 0 pour chat, 1 pour chien. La ground truth est la référence absolue utilisée pour calculer l'erreur du réseau.
Normalization / Preprocessing Données
→ Normalisation / Prétraitement
Transformation des données brutes avant de les donner au réseau. Dans notre TP : diviser par 255 pour avoir des valeurs entre 0 et 1. Essentiel pour la stabilité du gradient — évite que certaines features dominent les autres.
Class imbalance Données
→ Déséquilibre des classes
Situation où certaines classes sont beaucoup plus représentées que d'autres. Ex. : 95% de chats, 5% de chiens. Le modèle peut atteindre 95% d'accuracy en prédisant toujours "chat". Nécessite un traitement spécifique.
Machine Learning IA générale
→ Apprentissage automatique
Sous-domaine de l'IA où les algorithmes apprennent à partir de données sans être explicitement programmés. Trois grandes familles : supervisé (avec labels), non supervisé (sans labels), par renforcement (avec récompenses).
Deep Learning IA générale
→ Apprentissage profond
Sous-domaine du Machine Learning utilisant des réseaux de neurones à de nombreuses couches (profondes). La profondeur permet d'apprendre des représentations hiérarchiques : pixels → bords → formes → objets.
Inference IA générale
→ Inférence
Phase d'utilisation du modèle après entraînement : on lui donne une nouvelle donnée et il produit une prédiction. Différent de l'entraînement (pas de calcul de gradient ni de mise à jour des poids).
Classification vs Regression IA générale
→ Classification vs Régression
Classification : prédit une catégorie (chat ou chien, spam ou non). Régression : prédit une valeur continue (prix d'une maison, température demain). Notre TP est une classification binaire.
RNN / LSTM / GRU IA générale
→ Réseau de neurones récurrent
Architectures conçues pour les données séquentielles (texte, son, séries temporelles). Contrairement au MLP et CNN, elles ont une mémoire interne qui persiste d'un élément à l'autre de la séquence.
Transformer / LLM IA générale
→ Transformer / Grand modèle de langage
Architecture révolutionnaire (2017) basée sur l'attention. Remplace les RNN pour le NLP. Base de GPT, BERT, Claude, LLaMA. Les LLM (Large Language Models) sont des Transformers entraînés sur des milliards de textes.
GAN IA générale
→ Réseau antagoniste génératif
Deux réseaux en compétition : un générateur crée de fausses images, un discriminateur essaie de les distinguer des vraies. Au fil de l'entraînement, le générateur s'améliore jusqu'à produire des images photoréalistes.
GPU vs CPU IA générale
→ Processeur graphique vs processeur central
Le GPU (carte graphique) excelle pour le deep learning car il effectue des milliers d'opérations matricielles en parallèle. Un GPU peut entraîner un CNN 10 à 100 fois plus vite qu'un CPU pour des images.
Fine-tuning IA générale
→ Ajustement fin
Ré-entraînement d'un modèle pré-entraîné sur un nouveau dataset spécifique, avec un très faible learning rate. Les premières couches (features génériques) sont souvent gelées, seules les dernières couches s'adaptent au nouveau domaine.
Benchmark / ImageNet IA générale
→ Référence / Jeu de test standard
Dataset ou tâche standard servant à comparer les modèles. MNIST : 60 000 chiffres manuscrits. CIFAR-10 : 60 000 images 32×32. ImageNet : 1,2 million d'images, 1000 classes. Référence absolue du deep learning vision.
