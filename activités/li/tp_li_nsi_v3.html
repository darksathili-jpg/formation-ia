01 Présentation du projet Li
Qu'est-ce que Li ? Un assistant vocal entièrement autonome

Li est un agent vocal à intelligence artificielle qui fonctionne entièrement sur un Raspberry Pi 5, sans aucune connexion à internet. Il écoute votre voix, comprend vos questions, génère une réponse grâce à un modèle de langage exécuté localement et vous répond à voix haute, avec un visage animé qui exprime ses émotions en temps réel.

Ce projet couvre six domaines du programme de Terminale NSI : la programmation orientée objet, le traitement du signal audio, la concurrence et le parallélisme, les architectures matérielles, les API REST et les systèmes d'exploitation.

Objectif du TP Construire Li de zéro en suivant ce document. À la fin, votre Raspberry Pi 5 dans son boîtier Freenove répondra vocalement à vos questions, en français, hors ligne, avec une latence totale inférieure à 12 secondes.

02 Architecture logicielle
Décomposition modulaire Quatre fichiers Python · un rôle par module
li_main.py

Orchestrateur principal. Initialise tous les sous-systèmes, gère la mémoire conversationnelle, coordonne les appels à Whisper, Ollama et Piper. Contient les méthodes _record() et _synthese().

li_face.py

Rendu pygame à 30 FPS. Gère les sept états émotionnels et l'animation de la bouche synchronisée sur le niveau RMS du signal audio.

li_wake.py

Écoute continue du microphone. Détecte le mot-clé par fenêtres glissantes de 80 ms via openWakeWord. Lance le callback _on_wake dans un thread séparé.

li_audio.py

Lecture du fichier WAV produit par Piper. Calcule le niveau RMS par tranches de 50 ms et le transmet au visage pour animer la bouche.

Modèle de threads Trois threads permanents · concurrence · sections critiques

Li exécute trois threads en parallèle, ce qui permet au visage de rester toujours animé pendant les traitements lourds (transcription, inférence LLM).

Thread Rôle Cadence Ressource partagée
T-Wake Écoute du micro, détection mot-clé, lancement T-Pipeline Continu Aucune
T-Pipeline Transcription → LLM → synthèse → lecture audio À la demande Écriture sur _emotion, _mouth_open
T-Face Rendu pygame à 30 images par seconde 30 Hz Lecture de _emotion, _mouth_open
Conception importante — T-Wake ne bloque pas Dans la version correcte du code, lorsque le mot-clé est détecté, T-Wake lance T-Pipeline dans un nouveau thread (threading.Thread(target=self._on_wake).start()) et reprend immédiatement son écoute. Si T-Wake appelait directement _on_wake(), il serait bloqué pendant 10 secondes et ne pourrait pas détecter un second appel. Cette distinction est architecturalement critique.

Démarrage depuis le SSD NVMe Bootloader · PCIe · ordre de boot

Par défaut, le Raspberry Pi 5 démarre depuis la carte microSD. Le M.2 HAT du FNK0100 connecte le SSD NVMe via le bus PCIe Gen 2 interne. Pour démarrer depuis le SSD (lectures ~900 Mo/s contre ~45 Mo/s pour la microSD), il faut modifier l'ordre de boot dans l'EEPROM.

Procédure de configuration du bootloader Depuis un terminal sur le RPi (avec microSD en cours d'utilisation) :
# Éditer la configuration du bootloader EEPROM sudo rpi-eeprom-config --edit # Dans l'éditeur, modifier ou ajouter la ligne : BOOT_ORDER=0xf416 # 6 = NVMe · 1 = SD · 4 = USB · f = fin de liste # L'ordre de lecture est de droite à gauche : NVMe en premier # Appliquer et redémarrer sudo reboot

#!/usr/bin/env python3 # li_main.py — Orchestrateur principal de Li # Terminale NSI · Projet IA Embarquée · Raspberry Pi 5 import threading import time import tempfile import subprocess import os import numpy as np import pyaudio import ollama from faster_whisper import WhisperModel from li_face import LiFace, Emotion from li_wake import LiWakeWord from li_audio import LiAudio # ── Configuration centralisée ───────────────────────────────────────────── CONFIG = { "model" : "llama3.2:3b", # modèle Ollama "whisper_model" : "small", # taille du modèle Whisper "lang" : "fr", # langue de transcription "samplerate" : 16000, # Hz — imposé par openWakeWord "listen_secs" : 6, # durée d'enregistrement après wake "piper_bin" : "../models/piper/piper/piper", "piper_model" : "../models/piper/fr_FR-upmc-medium.onnx", "system_prompt" : ( "Tu es Li, un assistant vocal bienveillant, concis et précis. " "Réponds TOUJOURS en français. Limite chaque réponse à 2 phrases " "maximum pour rester adapté à la synthèse vocale." ), } class LiAssistant: """Orchestrateur principal de Li — gère le pipeline complet.""" def __init__(self): print("[Li] Chargement de Whisper ({}, int8)...".format(CONFIG["whisper_model"])) t0 = time.time() self.whisper = WhisperModel( CONFIG["whisper_model"], compute_type="int8" ) print("[Li] Whisper prêt — {:.1f} s".format(time.time() - t0)) self.face = LiFace(width=800, height=480) # résolution écran FNK0100 self.wake = LiWakeWord(callback=self._on_wake_threaded) self.audio = LiAudio() # Mémoire conversationnelle : system_prompt + max 20 derniers messages self.history = [ {"role": "system", "content": CONFIG["system_prompt"]} ] # Verrou pour éviter deux pipelines simultanés self._pipeline_lock = threading.Lock() print("[Li] Prêt. Dites 'Bonjour Li' pour commencer.") # ── Callback appelé par T-Wake ──────────────────────────────────────── def _on_wake_threaded(self): """Lance le pipeline dans un thread séparé pour ne pas bloquer T-Wake.""" if self._pipeline_lock.locked(): # Li est déjà en train de traiter une requête — ignorer return threading.Thread(target=self._pipeline, daemon=True).start() # ── Pipeline principal ──────────────────────────────────────────────── def _pipeline(self): """Étapes : écoute → STT → LLM → TTS → lecture.""" with self._pipeline_lock: # Étape 1 — Enregistrement self.face.set_emotion(Emotion.LISTENING) audio_np = self._record(CONFIG["listen_secs"]) # Étape 2 — Transcription Whisper (STT) self.face.set_emotion(Emotion.THINKING) segments, _ = self.whisper.transcribe( audio_np, language=CONFIG["lang"], vad_filter=True # supprime les silences ) texte = " ".join(s.text for s in segments).strip() if not texte: print("[Li] Rien entendu.") self.face.set_emotion(Emotion.IDLE) return print("[Li] Transcription : {}".format(texte)) # Étape 3 — Requête LLM (Ollama) self.history.append({"role": "user", "content": texte}) try: resp = ollama.chat( model=CONFIG["model"], messages=self.history ) reponse = resp["message"]["content"].strip() except Exception as e: reponse = "Désolé, je n'arrive pas à accéder au modèle." print("[Li] Erreur Ollama :", e) self.history.append({"role": "assistant", "content": reponse}) self._tronquer_historique() print("[Li] Réponse : {}".format(reponse)) # Étape 4 — Synthèse vocale (TTS Piper) wav_path = self._synthese(reponse) if wav_path: self.audio.play_wav( wav_path, face_cb=lambda lvl: self.face.set_speaking(True, lvl) ) os.unlink(wav_path) # supprimer le fichier temporaire self.face.set_emotion(Emotion.IDLE) # ── Enregistrement microphone ───────────────────────────────────────── def _record(self, duree_s: float) -> np.ndarray: """ Enregistre duree_s secondes depuis le microphone. Retourne un tableau numpy float32 normalisé entre -1.0 et 1.0, attendu par faster-whisper.transcribe(). """ CHUNK = 1024 SR = CONFIG["samplerate"] n_frames = int(SR * duree_s) pa = pyaudio.PyAudio() stream = pa.open( rate=SR, channels=1, format=pyaudio.paInt16, input=True, frames_per_buffer=CHUNK ) frames = [] collected = 0 while collected < n_frames: data=stream.read(CHUNK, exception_on_overflow=False) frames.append(data) collected += CHUNK stream.stop_stream() stream.close() pa.terminate() # Convertir bytes → int16 → float32 normalisé raw = b"".join(frames) audio_int16 = np.frombuffer(raw, dtype=np.int16) audio_float32 = audio_int16.astype(np.float32) / 32768.0 return audio_float32 # ── Synthèse vocale Piper TTS ───────────────────────────────────────── def _synthese(self, texte: str) -> str or None: """ Appelle le binaire Piper en sous-processus. Retourne le chemin vers le fichier WAV généré, ou None en cas d'erreur. """ try: fd, wav_path = tempfile.mkstemp(suffix=".wav") os.close(fd) cmd = [ os.path.expanduser(CONFIG["piper_bin"]), "--model", os.path.expanduser(CONFIG["piper_model"]), "--output_file", wav_path, ] proc = subprocess.run( cmd, input=texte.encode("utf-8"), capture_output=True, timeout=15 ) if proc.returncode != 0: print("[Li] Erreur Piper :", proc.stderr.decode()) return None return wav_path except Exception as e: print("[Li] Erreur synthèse :", e) return None # ── Gestion de l'historique ─────────────────────────────────────────── def _tronquer_historique(self): if len(self.history) > 21: # 1 system + 20 messages self.history = [self.history[0]] + self.history[-20:] # ── Démarrage ───────────────────────────────────────────────────────── def run(self): """Lance le thread de rendu du visage et le détecteur de mot-clé.""" face_thread = threading.Thread(target=self.face.run, daemon=True) face_thread.start() self.wake.start() try: while True: time.sleep(1) except KeyboardInterrupt: print("\n[Li] Arrêt demandé.") if __name__ == "__main__": li = LiAssistant() li.run()

#!/usr/bin/env python3 # li_face.py — Rendu du visage animé de Li via pygame import pygame import threading import math from enum import Enum, auto class Emotion(Enum): IDLE = auto() # repos — regard neutre LISTENING = auto() # écoute active THINKING = auto() # traitement en cours SPEAKING = auto() # synthèse vocale en cours HAPPY = auto() # réponse enthousiaste SURPRISED = auto() # question inattendue SLEEPING = auto() # veille (aucune activité prolongée) class LiFace: """ Rendu pygame du visage de Li. Thread-safe : tous les attributs partagés sont protégés par _lock. À exécuter dans un thread dédié via la méthode run(). """ def __init__(self, width: int = 320, height: int = 320): self._emotion = Emotion.IDLE self._mouth_open = 0.0 # 0.0 = fermée · 1.0 = grande ouverte self._running = False self._lock = threading.Lock() self.width = width self.height = height # Iris wandering — interpolation vers position cible self._iris_x = 0.0 # position actuelle (normalisée -1..1) self._iris_y = 0.0 self._iris_tx = 0.0 # position cible self._iris_ty = 0.0 self._next_wander = 0.0 # ── Interface publique (thread-safe) ────────────────────────────────── def set_emotion(self, emotion: Emotion): with self._lock: self._emotion = emotion self._mouth_open = 0.0 def set_speaking(self, speaking: bool, audio_level: float = 0.0): with self._lock: self._emotion = Emotion.SPEAKING if speaking else Emotion.IDLE self._mouth_open = max(0.0, min(1.0, audio_level)) # ── Dessin du visage ────────────────────────────────────────────────── def _update_iris_wander(self, now: float): """Met à jour la position du regard par interpolation exponentielle.""" import random if now >= self._next_wander: self._iris_tx = random.uniform(-0.4, 0.4) self._iris_ty = random.uniform(-0.4, 0.4) self._next_wander = now + random.uniform(1.5, 4.0) # Interpolation douce (easing exponentiel) self._iris_x += (self._iris_tx - self._iris_x) * 0.08 self._iris_y += (self._iris_ty - self._iris_y) * 0.08 def _draw(self, surface, now: float): """Dessine le visage complet selon l'état émotionnel actuel.""" with self._lock: emotion = self._emotion mouth_open = self._mouth_open W, H = self.width, self.height cx, cy = W // 2, H // 2 eye_y = cy - eye_r - 6), (ex + eye_r * s, eye_y - eye_r - 14), 3 ) if joues: # Joues roses semi-transparentes s = pygame.Surface((eye_r*3, eye_r), pygame.SRCALPHA) pygame.draw.ellipse(s, (244,114,182,65), (0,0, eye_r*3, eye_r)) surface.blit(s, (cx - mouth_w//2, mouth_y, mouth_w, max(4, mouth_h))) if mouth_open > 0.08: pygame.draw.ellipse(surface, (232,240,255), (cx - int(mouth_w*0.7)//2, mouth_y, int(mouth_w*0.7), int(max(2, mouth_h)*0.4))) # ── Boucle principale 30 FPS ────────────────────────────────────────── def run(self): """À lancer dans un thread dédié. #!/usr/bin/env python3 # li_wake.py — Détection du mot-clé via openWakeWord import pyaudio import numpy as np import time import threading from openwakeword.model import Model SAMPLERATE = 16_000 # Hz — imposé par openWakeWord CHUNK = 1_280 # = 80 ms par fenêtre (1280 / 16000) THRESHOLD = 0.70 # score minimal pour un déclenchement COOLDOWN_S = 2.0 # secondes d'attente après chaque déclenchement class LiWakeWord: """ Écoute continue du microphone à 16 kHz. Détecte le mot-clé par fenêtres glissantes de 80 ms. Appelle callback() dans le thread de T-Wake lorsque détecté. """ def __init__(self, callback): self.callback = callback self._last_trigger = 0.0 print("[Li] Chargement du modèle openWakeWord...") self.model = Model( wakeword_models=["hey_jarvis"], inference_framework="onnx" ) print("[Li] openWakeWord prêt — modèle : hey_jarvis") def _listen_loop(self): """Boucle d'écoute continue — s'exécute dans T-Wake.""" pa = pyaudio.PyAudio() stream = pa.open( rate=SAMPLERATE, channels=1, format=pyaudio.paInt16, input=True, frames_per_buffer=CHUNK ) try: while True: raw = stream.read(CHUNK, exception_on_overflow=False) audio = np.frombuffer(raw, dtype=np.int16) scores = self.model.predict(audio) score = max(scores.values(), default=0.0) maintenant = time.time() if (score >= THRESHOLD and (maintenant - self._last_trigger) > COOLDOWN_S): self._last_trigger = maintenant print("[Li] Mot-clé détecté (score={:.2f})".format(score)) self.callback() # appelle _on_wake_threaded finally: stream.stop_stream() stream.close() pa.terminate() def start(self): """Démarre T-Wake en daemon thread.""" t = threading.Thread(target=self._listen_loop, daemon=True) t.start() print("[Li] T-Wake démarré. Dites 'hey jarvis'.")

RMS = √( (1/N) ⋅ ∑ xᵢ² ) xᵢ : échantillon normalisé entre −1 et 1  ·  N : nombre d'échantillons dans la fenêtre

#!/usr/bin/env python3 # li_audio.py — Lecture du WAV et calcul RMS pour la synchronisation labiale import wave import subprocess import time import numpy as np CHUNK_MS = 50 # taille de la fenêtre d'analyse RMS (millisecondes) class LiAudio: """ Gère la lecture audio et la synchronisation labiale de Li. play_wav() lit un fichier WAV par tranches de CHUNK_MS ms, calcule le niveau RMS de chaque tranche et le transmet au visage via un callback pour animer la bouche en synchronisation avec la voix. """ def calculer_rms(self, samples_int16: np.ndarray) -> float: """ Calcule le niveau RMS normalisé entre 0.0 et 1.0. Le facteur ×4 amplifie le signal (la voix TTS est relativement douce). """ if len(samples_int16) == 0: return 0.0 MAX_VAL = 32768.0 samples = samples_int16.astype(np.float32) / MAX_VAL rms = np.sqrt(np.mean(samples ** 2)) return float(np.clip(rms * 4.0, 0.0, 1.0)) def play_wav(self, path: str, face_cb): """ Lit path via aplay (sortie ALSA) et analyse le RMS en parallèle. face_cb(niveau) est appelé toutes les CHUNK_MS ms pour piloter la bouche. Synchronisation : aplay et la boucle WAV démarrent simultanément. La boucle lit les mêmes frames que aplay joue, avec un délai CHUNK_MS. """ with wave.open(path, "rb") as wf: sr = wf.getframerate() chunk_size = sr * CHUNK_MS // 1000 # en nombre d'échantillons # Lancer aplay en arrière-plan proc = subprocess.Popen( ["aplay", "-q", path], stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL ) # Analyser le fichier WAV tranche par tranche while True: raw = wf.readframes(chunk_size) if not raw: break samples = np.frombuffer(raw, dtype=np.int16) niveau = self.calculer_rms(samples) face_cb(niveau) time.sleep(CHUNK_MS / 1000.0) proc.wait() # attendre la fin de aplay face_cb(0.0) # fermer la bouche
