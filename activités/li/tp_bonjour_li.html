01 Pourquoi entraîner son propre mot-clé ?
Le problème du mot-clé anglais Limite des modèles pré-entraînés

Dans le TP principal, Li se déclenche sur « hey jarvis », un mot-clé anglais. Cette solution fonctionne, mais elle présente deux limitations majeures pour un assistant vocal francophone.

Problème 1 — Expérience utilisateur

Un assistant vocal français qui répond au nom anglais « hey jarvis » manque de cohérence. L'utilisateur doit adopter une prononciation étrangère peu naturelle.

Problème 2 — Faux positifs

Le modèle hey_jarvis a été entraîné sur des voix anglophones. Avec une voix française, le taux de faux positifs (déclenchements non voulus) augmente.

La solution — entraîner un modèle personnalisé openWakeWord permet d'entraîner un modèle ONNX léger (~400 Ko) pour n'importe quel mot-clé, en utilisant de la synthèse vocale TTS pour générer automatiquement les données d'entraînement. Piper (déjà installé sur le RPi) produit ces données.

Le format ONNX Open Neural Network Exchange — standard d'interopérabilité

ONNX est un format ouvert de représentation des modèles de réseaux de neurones, créé par Microsoft et Facebook en 2017. Il permet d'entraîner un modèle dans un framework (PyTorch, TensorFlow, scikit-learn) et de l'exécuter dans un autre.

Entraînement → PyTorch

Framework riche, flexible, adapté à la recherche. Tourne bien sur PC avec ou sans GPU. Syntaxe Python naturelle.

Inférence → ONNX Runtime

Moteur d'exécution optimisé, léger (~5 Mo). Fonctionne sur ARM64 sans PyTorch. Idéal pour le RPi embarqué.

Fonction de perte et entraînement BCE · gradient · epoch · batch

L'entraînement consiste à ajuster les poids de la tête de classification pour minimiser une fonction de perte (loss). Pour un problème de classification binaire (présence ou absence du mot-clé), on utilise la Binary Cross-Entropy.

BCE = − [ y · log(ŷ) + (1−y) · log(1−ŷ) ] y : étiquette réelle (0 ou 1)  ·  ŷ : score prédit par le modèle (0.0 à 1.0)
Epoch

Un passage complet sur l'ensemble des données d'entraînement. Après chaque epoch, la loss devrait diminuer.

Batch

Sous-ensemble des données traité à chaque mise à jour des poids. Compromis entre précision du gradient et vitesse.

Taux d'apprentissage

Amplitude du pas lors de la descente de gradient. Trop grand : oscillations. Trop petit : convergence lente.

03 Génération des données d'entraînement
Principe — apprentissage par synthèse Piper TTS · augmentation · étiquetage automatique

openWakeWord adopte une approche originale : plutôt que d'enregistrer des centaines de personnes différentes, on utilise la synthèse vocale TTS pour générer automatiquement des milliers de variations du mot-clé. Piper, déjà installé sur le RPi, produit ces fichiers.

Composition du jeu de données Un bon jeu d'entraînement contient deux types d'exemples équilibrés : les positifs (occurrences du mot-clé « Bonjour Li ») et les négatifs (tout le reste : parole ordinaire, bruits, mots phonétiquement proches). Sans négatifs, le modèle prédit toujours 1.

#!/usr/bin/env python3 # generer_donnees.py — Génération du jeu de données pour "Bonjour Li" # À exécuter sur le Raspberry Pi 5 dans ~/li-project/src/ import subprocess, os, random, struct, wave import numpy as np from pathlib import Path # ── Configuration ──────────────────────────────────────────────────────── PIPER_BIN = Path("../models/piper/piper/piper") PIPER_MODEL = Path("../models/piper/fr_FR-upmc-medium.onnx") DATA_DIR = Path("../data") SR = 16000 # Hz — fréquence imposée par openWakeWord # Phrases positives — variations linguistiques de "Bonjour Li" POSITIFS = [ "Bonjour Li", "Bonjour Li.", "Bonjour, Li.", "Bonjour Li !", ] # Phrases négatives — parole ordinaire et distracteurs phonétiques NEGATIFS_PAROLE = [ "Bonjour tout le monde", "Bonsoir Li", "Au revoir Li", "Bonjour Lou", "Bonjour Lili", "Allô Li", "Quelle heure est-il", "Quel temps fait-il aujourd'hui", "Dis-moi une blague", "Merci beaucoup", "Python est un langage de programmation", "Le Raspberry Pi 5 dispose de quatre cœurs", "L'intelligence artificielle transforme notre quotidien", "Ouvre la porte", "Éteins la lumière", "Je voudrais commander une pizza", "Raconte-moi une histoire", ] def synthese_wav(texte: str, chemin_sortie: Path, speed: float = 1.0, pitch_offset: int = 0): """Génère un fichier WAV 16 kHz mono via Piper TTS.""" cmd = [ str(PIPER_BIN.expanduser()), "--model", str(PIPER_MODEL.expanduser()), "--output_file", str(chemin_sortie), "--length_scale", str(1.0 / max(speed, 0.1)), ] subprocess.run(cmd, input=texte.encode("utf-8"), capture_output=True, timeout=20) def bruit_blanc_wav(chemin_sortie: Path, duree_s: float = 1.0, amplitude: float = 0.05): """Génère un fichier WAV de bruit blanc gaussien.""" n = int(SR * duree_s) samples = (np.random.randn(n) * amplitude * 32767).astype(np.int16) with wave.open(str(chemin_sortie), "wb") as wf: wf.setnchannels(1) wf.setsampwidth(2) wf.setframerate(SR) wf.writeframes(samples.tobytes()) def generer(): # Créer l'arborescence (DATA_DIR / "positifs").mkdir(parents=True, exist_ok=True) (DATA_DIR / "negatifs").mkdir(parents=True, exist_ok=True) total_pos = 0 total_neg = 0 # ── Positifs : 250 variations par phrase × 4 phrases = 1 000 ───────── print("[1/3] Génération des positifs...") speeds = [0.80, 0.90, 1.00, 1.05, 1.10] for idx_phrase, phrase in enumerate(POSITIFS): for rep in range(250): speed = random.choice(speeds) + random.uniform(-0.05, 0.05) nom = DATA_DIR / "positifs" / f"pos_{idx_phrase:02d}_{rep:04d}.wav" synthese_wav(phrase, nom, speed=speed) total_pos += 1 print(f" '{phrase}' : {total_pos} fichiers générés") # ── Négatifs parole : 100 répétitions × 17 phrases = 1 700 ────────── print("[2/3] Génération des négatifs (parole)...") for idx_phrase, phrase in enumerate(NEGATIFS_PAROLE): for rep in range(100): speed = random.uniform(0.85, 1.15) nom = DATA_DIR / "negatifs" / f"neg_p_{idx_phrase:02d}_{rep:04d}.wav" synthese_wav(phrase, nom, speed=speed) total_neg += 1 print(f" {total_neg} fichiers négatifs (parole) générés") # ── Négatifs bruit : 500 fichiers de bruit blanc ───────────────────── print("[3/3] Génération des négatifs (bruit)...") for i in range(500): amplitude = random.uniform(0.01, 0.15) duree = random.uniform(0.5, 2.0) nom = DATA_DIR / "negatifs" / f"neg_b_{i:04d}.wav" bruit_blanc_wav(nom, duree_s=duree, amplitude=amplitude) total_neg += 1 print(f"\n[OK] Génération terminée.") print(f" Positifs : {total_pos}") print(f" Négatifs : {total_neg}") print(f" Total : {total_pos + total_neg}") print(f" Dossier : {DATA_DIR.resolve()}") if __name__ == "__main__": generer()

#!/usr/bin/env python3 # train_bonjour_li.py — Entraîne la tête de classification openWakeWord # À exécuter sur le PC de salle dans li-train-env import os, random, time import numpy as np import torch import torch.nn as nn from torch.utils.data import DataLoader, TensorDataset from pathlib import Path import librosa from tqdm import tqdm import torch.onnx # ── Hyperparamètres ─────────────────────────────────────────────────────── EPOCHS = 30 BATCH_SIZE = 64 LR = 1e-3 VAL_SPLIT = 0.15 # 15 % des données réservées à la validation DATA_DIR = Path("data") SR = 16000 N_MELS = 32 # bandes du spectrogramme de Mel EMBED_DIM = 96 # dimension des embeddings openWakeWord # ── Extraction des features ─────────────────────────────────────────────── def wav_vers_embedding(chemin: Path) -> np.ndarray: """ Charge un WAV, calcule son spectrogramme de Mel, et produit un embedding simplifié de dimension EMBED_DIM. Note : en production, openWakeWord utilise le modèle Google Audio Embedding pré-entraîné. Ici, on utilise une approximation par Mel + statistiques. """ try: y, _ = librosa.load(str(chemin), sr=SR, mono=True, duration=1.5) if len(y) < SR // 4: return None mel = librosa.feature.melspectrogram(y=y, sr=SR, n_mels=N_MELS) mel_db = librosa.power_to_db(mel, ref=np.max) # Statistiques sur le spectrogramme → vecteur EMBED_DIM mu = np.mean(mel_db, axis=1) # 32 valeurs sigma = np.std(mel_db, axis=1) # 32 valeurs dyn = np.percentile(mel_db, [25, 50, 75], axis=1).mean(axis=1) feat = np.concatenate([mu, sigma, dyn])[:EMBED_DIM] return feat.astype(np.float32) except: return None def charger_donnees(): X, y = [], [] print("Chargement et extraction des features...") for fichier in tqdm(list((DATA_DIR / "positifs").glob("*.wav"))): emb = wav_vers_embedding(fichier) if emb is not None: X.append(emb); y.append(1.0) for fichier in tqdm(list((DATA_DIR / "negatifs").glob("*.wav"))): emb = wav_vers_embedding(fichier) if emb is not None: X.append(emb); y.append(0.0) X = np.array(X, dtype=np.float32) y = np.array(y, dtype=np.float32) # Mélanger idx = np.random.permutation(len(X)) return X[idx], y[idx] # ── Architecture de la tête de classification ───────────────────────────── class TeteClassification(nn.Module): """ Réseau dense léger entraînable. 96 → Dense(32, ReLU) → Dense(16, ReLU) → Dense(1, Sigmoid) """ def __init__(self): super().__init__() self.reseau = nn.Sequential( nn.Linear(EMBED_DIM, 32), nn.ReLU(), nn.Dropout(0.3), # regularisation contre le surapprentissage nn.Linear(32, 16), nn.ReLU(), nn.Linear(16, 1), nn.Sigmoid() ) def forward(self, x): return self.reseau(x).squeeze(1) # ── Boucle d'entraînement ───────────────────────────────────────────────── def entrainer(): X, y = charger_donnees() n_val = int(len(X) * VAL_SPLIT) X_tr, X_va = X[n_val:], X[:n_val] y_tr, y_va = y[n_val:], y[:n_val] ds_train = DataLoader(TensorDataset(torch.tensor(X_tr), torch.tensor(y_tr)), batch_size=BATCH_SIZE, shuffle=True) ds_val = DataLoader(TensorDataset(torch.tensor(X_va), torch.tensor(y_va)), batch_size=BATCH_SIZE) modele = TeteClassification() criterion = nn.BCELoss() optimizer = torch.optim.Adam(modele.parameters(), lr=LR) scheduler = torch.optim.lr_scheduler.StepLR(optimizer, step_size=10, gamma=0.5) meilleure_val_loss = float("inf") meilleur_epoch = 0 print(f"\n{'Epoch':>5} | {'Train loss':>11} | {'Val loss':>10} | {'Val acc':>8}") print("-" * 46) for ep in range(1, EPOCHS + 1): # Phase d'entraînement modele.train() loss_train = 0.0 for xb, yb in ds_train: optimizer.zero_grad() pred = modele(xb) loss = criterion(pred, yb) loss.backward() optimizer.step() loss_train += loss.item() * len(xb) loss_train /= len(X_tr) # Phase de validation modele.eval() loss_val = 0.0; correct = 0 with torch.no_grad(): for xb, yb in ds_val: pred = modele(xb) loss_val += criterion(pred, yb).item() * len(xb) correct += ((pred >= 0.5) == (yb >= 0.5)).sum().item() loss_val /= len(X_va) acc_val = correct / len(X_va) * 100 scheduler.step() etoile = " *" if loss_val < meilleure_val_loss else "" if loss_val < meilleure_val_loss: meilleure_val_loss=loss_val meilleur_epoch=ep torch.save(modele.state_dict(), "meilleur_modele.pt") print(f"{ep:>5} | {loss_train:>11.5f} | {loss_val:>10.5f} | {acc_val:>7.2f}%{etoile}") print(f"\nMeilleur modèle : epoch {meilleur_epoch} · val loss {meilleure_val_loss:.5f}") return modele # ── Export ONNX ─────────────────────────────────────────────────────────── def exporter_onnx(modele): modele.load_state_dict(torch.load("meilleur_modele.pt")) modele.eval() dummy = torch.zeros(1, EMBED_DIM) torch.onnx.export( modele, dummy, "bonjour_li.onnx", input_names=["embedding"], output_names=["score"], dynamic_axes={"embedding": {0: "batch"}}, opset_version=11 ) print("Export ONNX : bonjour_li.onnx") if __name__ == "__main__": modele = entrainer() exporter_onnx(modele)

05 Évaluation du modèle
La matrice de confusion Quatre cas possibles pour un classifieur binaire

Pour évaluer un détecteur de mot-clé, il ne suffit pas de compter les bonnes réponses. Il faut distinguer les quatre cas possibles selon la réalité et la prédiction.

Prédit : MOT-CLÉ (1) Prédit : NÉGATIF (0)
Réel : MOT-CLÉ (1) Vrai Positif (VP)
Li se déclenche correctement		 Faux Négatif (FN)
Li ne répond pas — raté
Réel : NÉGATIF (0) Faux Positif (FP)
Li se déclenche sans raison		 Vrai Négatif (VN)
Li reste silencieux — correct
Métriques d'évaluation Précision · Rappel · F1-score
Précision
VP / (VP + FP)
Parmi tous les déclenchements, combien étaient justifiés ?
Haute précision = peu de faux positifs.
Rappel
VP / (VP + FN)
Parmi tous les vrais mots-clés, combien ont été détectés ?
Haut rappel = peu de ratés.
F1-score
2 · P · R / (P + R)
Moyenne harmonique entre précision et rappel.
Équilibre les deux objectifs.
