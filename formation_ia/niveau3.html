01

Comment fonctionne un transformer ?

Le mécanisme derrière Claude, ChatGPT et Mistral — sans les mathématiques

1

Tokenisation

Le texte est découpé en unités lexicales (tokens). Chaque token est converti en un entier unique.

"Dijkstra" → ["Di","jk","str","a"]
"BFS" → ["B","FS"]
"récursivité" → ["r","éc","urs","ivit","é"]

Un mot-clé Python = 1–3 tokens. Un programme de 300 lignes ≈ 3 000–5 000 tokens.

2

Embeddings

Chaque token est représenté par un vecteur de haute dimension (768 à 4 096 valeurs).

Token → Vecteur [768 dim]
Capture le sens sémantique
"roi" - "homme" + "femme" ≈ "reine"

Deux tokens proches dans l'espace vectoriel ont un sens proche. C'est pourquoi l'IA comprend les synonymes.

3

Mécanisme d'attention

Pour chaque token, le modèle calcule sa relation avec tous les autres tokens du contexte.

"Quel token est lié à quel autre ?"
→ Matrice de pertinence
→ Multi-head (8 à 96 têtes)

C'est pourquoi un prompt précis sur Dijkstra donne un meilleur résultat qu'un prompt vague sur "les graphes".

4

Génération

Le token suivant le plus probable est sélectionné parmi 50 000+ candidats selon la température.

→ Distribution sur ~50 000 tokens
→ Température basse = déterministe
→ Température haute = créatif

Pour les exercices NSI : temperature basse (0.2). Pour la créativité pédagogique : temperature haute (0.8–1.0).

Règle pratique fenêtre de contexte : Claude Sonnet = 200 000 tokens max · GPT-4o = 128 000 tokens max. Si un TP dépasse 800 lignes, découpez en plusieurs prompts. Pour un projet sur 6 séances, utilisez le « summary prompting » : résumez le contexte en 3 lignes en début de chaque nouveau prompt.

02

Fine-tuning, RAG et agents IA

Techniques avancées accessibles sans expertise en machine learning — panorama pour NSI

Fine-tuning

Spécialisation

Principe

Réentraîner un modèle existant sur un corpus de données spécifiques pour le spécialiser sur un domaine ou un format particulier.

Usage NSI concret

Fine-tuner un modèle sur 200 corrigés NSI Terminale pour qu'il génère des corrections conformes au barème Bac et à la terminologie officielle du programme.

Coûteux en données et en calcul. Rarement pertinent pour un enseignant seul. Faisable via Ollama local pour expérimenter. À réserver aux projets collectifs académiques.

RAG

Retrieval-Augmented

Principe

L'IA interroge une base documentaire avant de répondre, en injectant automatiquement des extraits pertinents dans le prompt (retrieval + génération).

Usage NSI concret

Chatbot qui répond aux questions des élèves en s'appuyant sur le cours PDF que vous avez fourni — et uniquement sur lui. Réponses sourcées et fiables.

Solution gratuite pour commencer : NotebookLM (Google) permet un RAG sur vos propres documents sans code. Mistral + base vectorielle pour les usages avancés.

Agents IA

Autonomie partielle

Principe

IA qui enchaîne des actions de façon autonome : chercher de l'information, écrire du code, l'exécuter, analyser le résultat, corriger, relancer.

Usage NSI concret

Agent qui génère un exercice Python, l'exécute dans un sandbox sécurisé, vérifie les tests unitaires, puis corrige automatiquement si les tests échouent.

En développement rapide en 2025. Usage en classe à expérimenter prudemment. Excellent sujet de projet NSI avancé. Outils : Claude Code, n8n, LangChain.

Atelier pratique agents (45 min) : Concevoir un agent simple avec un LLM + deux outils (exécution Python + recherche). Étape 1 : définir la boucle ReAct (Reason + Act). Étape 2 : implémenter avec l'IA comme assistant. Étape 3 : analyser les risques en classe et rédiger une charte d'usage pour les élèves.

03

TP avancé — Les élèves utilisent l'IA pour déboguer

Séance type · Arbres binaires de recherche avec assistant IA · 70 minutes

1
Mise en situation 15 min
L'enseignant fournit un code Python d'ABR intentionnellement bogué contenant 5 bugs (indentation, cas limite, logique d'insertion, gestion du nœud null, condition de récursion). Les élèves analysent le code sans IA pendant 15 minutes et notent leurs hypothèses.
2
Dialogue avec l'IA 20 min
Consigne : « Soumettez le bug n°1 à l'IA. Expliquez-lui le contexte (langage, classe, objectif). Évaluez sa réponse : est-elle correcte ? Complète ? Que lui manque-t-il pour trouver les 4 autres bugs ? » Chaque binôme formule ses prompts de façon autonome.
3
Analyse critique 20 min
Chaque binôme complète un tableau : bugs correctement identifiés / manqués / inventés par l'IA. Discussion guidée : pourquoi l'IA hallucine-t-elle sur ce type d'erreur ? Lien avec la tokenisation et la fenêtre de contexte limitée.
4
Méta-réflexion 15 min
Questions finales : Quelle part du travail l'IA a-t-elle réellement faite ? Avez-vous appris quelque chose que vous n'auriez pas appris sans elle ? Dans quelles situations l'éviter pour progresser ? Construction d'une doctrine personnelle d'usage.
Projet filé — 3 séances

Visualiseur de graphes

S1 — Spécification : avec l'IA, l'élève rédige les specs fonctionnelles : structures de données, API publique, cas d'usage. L'IA suggère, l'élève décide et justifie.

S2 — Implémentation : l'élève code avec l'IA comme pair-reviewer. « Est-ce que ma fonction BFS est correcte ? Quels cas limites ai-je oubliés ? »

S3 — Optimisation et bilan : complexité réelle vs théorique, documentation des prompts utilisés, bilan : quelle part de crédit revient à l'IA ?

-- Extrait généré avec l'IA (S2) : class Graphe: def __init__(self): self.adj = {} # dict[str, list] def ajouter_arc(self, u, v, poids=1): if u not in self.adj: self.adj[u] = [] self.adj[u].append((v, poids)) def bfs(self, depart): visite = set() file = [depart] ordre = [] while file: sommet = file.pop(0) if sommet not in visite: visite.add(sommet) ordre.append(sommet) for voisin, _ in self.adj.get( sommet, []): file.append(voisin) return ordre
04

Éthique avancée — Biais algorithmiques en NSI

Analyser, identifier et enseigner les biais des systèmes IA en classe

Biais de confirmation

L'IA renforce les opinions dominantes présentes dans ses données. Exercice : poser la même question sur un algorithme de tri, changer le ton du prompt, observer les variations de réponse.

Biais de représentation culturelle

Les exemples NSI générés sont souvent en contexte occidental et anglophone. Tester avec un contexte géographique différent révèle des disparités significatives dans la qualité des exemples produits.

Biais de performance linguistique

L'IA est meilleure en Python qu'en OCaml ou Prolog. Implication directe : les ressources générées pour des langages moins représentés nécessitent une vérification plus rigoureuse.

Hallucination sélective

L'IA hallucine davantage sur les sujets peu représentés dans son corpus (algorithmes des années 1990, variantes peu connues) que sur les sujets populaires (BFS, tri rapide, RSA).

TP : Mettre un biais en évidence

-- Protocole expérimental élèves : Prompt A : "Génère un exercice sur le tri rapide pour lycéens." Prompt B : "Génère un exercice sur le tri rapide pour lycéens dans un contexte africain." Prompt C : "Génère le même exercice mais en supposant que les élèves ont déjà vu le tri fusion." -- Questions d'analyse : 1. Les 3 sorties sont-elles équivalentes en qualité ? 2. Identifier 2 biais observés 3. Comment les corriger dans le prompt ?

Ce TP peut être réalisé en 30 minutes. Il développe l'esprit critique des élèves sur les systèmes IA tout en restant ancré dans un contexte algorithmique NSI familier.

05

IA Act européen — Ce que l'enseignant NSI doit savoir

Régulation, classification des risques, implications concrètes pour l'éducation

Risque
inacceptable

Interdit dans l'Union européenne

Exemples NSI directement concernés : notation automatisée des élèves sans supervision humaine, profilage comportemental des mineurs, systèmes de scoring d'élèves non transparents, manipulation psychologique par IA.

Risque
élevé

Encadrement strict requis — Conformité, audit, transparence

Exemples NSI : outil d'orientation scolaire automatisé, scoring d'élèves influençant leur parcours, système de détection de plagiat par IA utilisé comme décision finale. Ces outils existent : l'enseignant doit savoir qu'ils entrent dans cette catégorie.

Risque
limité

Obligation de transparence — L'utilisateur doit savoir qu'il parle à une IA

Exemples NSI : chatbot éducatif répondant aux questions des élèves, assistant de révision IA, tuteur virtuel. La mention « vous interagissez avec un système d'IA » est légalement obligatoire dans tous ces cas.

Risque
minimal

Utilisation libre — Vigilance conseillée

Exemples NSI : générateur d'exercices assisté par l'enseignant, outil de création de ressources pédagogiques, assistance à la correction supervisée. Usage courant en formation et en classe — pas de contrainte légale supplémentaire.

Point clé pour l'enseignant : Un outil de détection automatique de plagiat qui prend seul la décision de signalement tombe en catégorie « risque élevé » de l'IA Act. L'enseignant NSI, en tant que futur référent, doit être capable d'identifier ces cas et d'alerter sa direction sur les outils utilisés dans l'établissement.

06

Projet final — Séquence pédagogique NSI complète

Travail individuel sur 3h30 · Livrable professionnel · Rendu dans les 3 semaines

1

Plan de séquence

6 séances détaillées : objectifs, compétences (programme officiel BO 2019), modalités, durée. Généré avec l'IA et validé par vous.

2

Cours complet

Document cours de 3–5 pages : notions, définitions, exemples, pseudo-code annoté. Prompts few-shot utilisés documentés.

3

TP noté 20 pts

Sujet TP avec fichier de données fourni (CSV ou Python incomplet), barème détaillé. Code testé en machine avant diffusion.

4

Éval formative

Quiz 10 questions auto-corrigé (Moodle ou papier), différencié A/B/C, avec corrigé enseignant et grille d'autoévaluation élèves.

5

Réflexivité IA

1 page : prompts clés utilisés, erreurs IA détectées, temps gagné estimé, limites rencontrées, recommandations pour collègues.

Critères d'évaluation du projet : rigueur algorithmique (code testé en machine), cohérence pédagogique (progression logique des 6 séances), qualité de la réflexivité IA (identification précise des erreurs et limites), et utilisabilité directe en classe (le livrable doit être prêt à distribuer sans modification majeure).

07

Après la formation — Devenir référent IA

Plan d'action 90 jours · Rôle et responsabilités dans l'établissement

Plan d'action 90 jours

Trois actions concrètes à mener dans votre établissement après la formation :

J 1–30 Organiser une démonstration de 30 minutes pour les collègues volontaires. Partager 3 prompts testés sur vos propres cours. Créer un dossier partagé de ressources sur l'ENT académique.
J 31–60 Déployer un premier projet NSI avec usage IA réflexif (type visualiseur de graphes). Documenter les résultats et les difficultés. Rédiger une charte d'usage IA pour la classe.
J 61–90 Animer le stage Niveau 1 pour les collègues de l'établissement. Rejoindre le réseau académique de référents IA. Partager une séquence complète sur Pixees ou l'ENT.

Rôle de référent établissement

Point de contact pour les questions sur l'IA en classe : RGPD, outils autorisés, cas d'usage pertinents, conformité IA Act. Lien avec le réseau académique IA. Veille technologique partagée avec l'équipe pédagogique.

Contribuer à la communauté NSI

Partager prompts, séquences et bilans sur Pixees.fr (site NSI officiel) ou l'ENT académique. La mutualisation est la force de la communauté NSI nationale. Chaque ressource partagée économise des heures à des dizaines de collègues.
