01

Techniques avancées de prompting

Few-shot et Chain-of-Thought — les deux leviers fondamentaux du niveau 2

01

Few-Shot Prompting

Inclure directement dans le prompt plusieurs exemples de la sortie attendue avant de formuler la vraie demande. Le modèle « calibre » son format et son niveau de détail sur ces exemples.

-- Schéma few-shot : "Voici 2 exercices modèles : [Ex 1] Contexte : algo BFS Énoncé : ... Corrigé Python : ... [Ex 2] Contexte : pile/file Énoncé : ... Corrigé Python : ... Génère un exercice identique sur les ARBRES BINAIRES."

Quand l'utiliser : chaque fois que le format de sortie doit être très précis — barème, pseudo-code, structure de TP. Le few-shot réduit de 60 % les itérations nécessaires.

02

Chain-of-Thought (CoT)

Demander au modèle de raisonner étape par étape avant de produire la sortie. Cette technique améliore significativement la cohérence des exercices et la rigueur algorithmique.

-- Schéma CoT : "Avant de générer l'exercice, raisonne étape par étape : 1. Identifie les prérequis algorithmiques du thème 2. Détermine le niveau adapté Terminale (première vue ?) 3. Prévois les erreurs classiques des élèves sur ce sujet Puis génère l'exercice."

Quand l'utiliser : pour les sujets complexes — algorithmes récursifs, démonstrations de complexité, séquences où l'ordre des notions est critique.

Combinaison optimale : few-shot + CoT donne les meilleurs résultats pour les séquences pédagogiques complètes. Le CoT assure la rigueur conceptuelle, le few-shot garantit le format livrable immédiatement. Exemple : « Raisonne étape par étape [CoT], puis génère au format suivant [few-shot] ».

02

Cas pratique — Séquence algorithmique complète

De l'introduction au TP évalué — Graphes et plus court chemin (Dijkstra)

1
Cours magistral 1h
"Génère un cours de 2 pages sur Dijkstra : définition, pseudo-code annoté, exemple graphe 5 nœuds, complexité O((V+E)logV) expliquée pour des lycéens sans base en théorie des graphes."
2
Exercices guidés 45 min
"Crée 3 exercices progressifs (facile / moyen / difficile) sur Dijkstra avec graphes ASCII, questions guidées et corrigés Python commentés utilisant heapq."
3
TP noté 1h30
"Génère un TP évalué : carte d'une ville (9 nœuds), fichier CSV fourni, script Python incomplet à compléter, barème détaillé 20 pts avec critères de correction."
4
Auto-évaluation 20 min
"Produis 5 questions flash de consolidation Dijkstra vs Bellman-Ford, format vrai/faux + justification courte, corrigé enseignant séparé."

Règle de découpage : Générez chaque composant (cours, TP, éval) dans des prompts séparés. Un TP de 300 lignes ≈ 4 000 tokens. Si votre séquence dépasse 800 lignes, découpez en plusieurs prompts et utilisez le « summary prompting » : en début de chaque nouveau prompt, résumez le contexte déjà généré en 2–3 lignes.

Prompts testés et validés — algorithmes, cybersécurité, réseaux, bases de données

Algorithmique

Structures de données — Piles

"Tu es prof NSI Terminale. Crée un TP complet sur les piles en Python : implémentation (push/pop/peek/isEmpty), vérification de parenthèses, exercice bonus pile avec deux files en 4 étapes."

  • Classe Stack commentée
  • Application vérification d'expressions
  • Exercice guidé complexité amortie

Complexité et récursivité

"Raisonne étape par étape. 1. Cas de base Fibonacci ? 2. Pile d'appels pour n=4 ? 3. Génère exercice récursif + mémoïsation. 4. Visualise l'arbre d'appels ASCII."

  • Arbre d'appels fib(4) en ASCII
  • Comparatif récursif / mémoïsé
  • Tableau complexités O(n) pour 6 algos

Cybersécurité

Cryptographie RSA

"Crée un exercice complet sur RSA : génère p=17, q=11, calcule n, φ(n), e, d. Guide l'élève pas à pas. Corrigé Python avec sympy. Niveau Terminale. Barème 12 pts."

  • Exercice dirigé 5 étapes
  • Code Python sympy complet
  • Barème détaillé 12 pts

Analyse de protocoles

"Génère un scénario d'analyse Wireshark simulée : HTTP sur TCP, 3-way handshake, fragmentation IP. Questions progressives avec captures ASCII annotées."

  • Trame ASCII commentée
  • 8 questions progressives
  • Corrigé complet

Réseaux et BDD

Schéma relationnel SQL

"Génère un schéma SQL bibliothèque scolaire : tables Livre, Élève, Emprunt, Auteur, clés primaires/étrangères. 3 requêtes progressives SELECT/JOIN/GROUP BY avec corrigés."

  • CREATE TABLE complet
  • 3 niveaux SELECT, JOIN, GROUP BY
  • Corrigé enseignant séparé

Adressage IP et routage

"Génère 5 exercices progressifs sur le découpage CIDR /24 → /26 : calcul de plages, sous-réseaux, représentation binaire. Script Python de vérification inclus."

  • 5 exercices + fichiers CSV
  • Script Python de vérification
  • Corrigé étape par étape
04

Différenciation pédagogique automatisée

Générer trois niveaux de ressources sur le même sujet — exemple : arbres binaires

Parcours A — Aide

Accompagnement renforcé

Prompt : « Génère le même exercice sur les arbres binaires mais avec : squelette de code Python à compléter, indices progressifs débloquables, explications inline sur chaque étape. »

  • Code Python préstructuré avec pass
  • Commentaires guidants détaillés
  • Tests unitaires fournis
  • Schéma visuel ASCII de l'arbre

Parcours B — Standard

Niveau programme

Prompt : « Exercice standard arbres binaires : insérer, parcourir en largeur, compter feuilles. Énoncé seul, barème 20 pts détaillé, corrigé enseignant séparé. »

  • 3 questions de difficulté croissante
  • Barème 20 pts précis
  • Corrigé enseignant complet
  • Grille d'autoévaluation

Parcours C — Approfondissement

Enrichissement avancé

Prompt : « Version avancée : arbre AVL, équilibrage automatique, démonstration de complexité O(log n). Introduire la notion de facteur d'équilibre et rotation simple. »

  • Exercice ouvert de recherche
  • Piste de recherche personnelle
  • Lien vers ressources avancées
  • Démonstration de la propriété AVL

Méthode de production : Générez d'abord le parcours B (standard). Puis demandez dans deux prompts distincts : « Adapte cet exercice pour le parcours A en ajoutant squelette et indices » et « Adapte pour un élève avancé en introduisant les arbres AVL ». Cette approche garantit la cohérence thématique entre les trois niveaux.

05

Vérifier les sorties IA — Processus en 4 étapes

Un protocole systématique avant toute diffusion aux élèves

Étape 1 — Exécuter le code Python

Toujours tester le code dans Thonny, un notebook Jupyter ou VSCode. L'IA produit des erreurs d'indentation, d'import manquant, de cas limites non traités. Fréquence observée : 1 bug sur 3 exercices générés en moyenne. Une erreur non détectée introduit une confusion en classe difficile à corriger.

Étape 2 — Vérifier la rigueur algorithmique

Contrôler : complexité annoncée (O(n) vs O(n²) ?), cas limites (liste vide, n=0, arbre null ?), cohérence avec le programme officiel NSI Terminale (BO 2019). Les LLM ont tendance à sous-estimer les complexités et à omettre les cas dégénérés.

Étape 3 — Tester la pertinence pédagogique

Une ressource techniquement correcte peut être pédagogiquement mal calibrée : trop facile, trop longue, vocabulaire inadapté. Faire relire par un collègue ou tester sur un groupe test avant diffusion à l'ensemble de la classe.

Étape 4 — Documenter et capitaliser

Conserver les prompts efficaces dans un fichier partagé (Notion, ENT, dossier partagé académique). Documenter les échecs est aussi précieux que les succès : un prompt qui a produit de mauvais résultats sur un thème particulier est une information utile pour toute la communauté NSI.

06

Atelier — Construire une séquence NSI de A à Z

Travail en binômes · 3h30 de production réelle · Livrable direct

1
30 min

Choix du thème et plan

Choisir un chapitre NSI Terminale. Générer avec l'IA un plan de séquence (4–6 séances). Identifier les prérequis algorithmiques et les erreurs élèves prévisibles.

2
1h30

Production des ressources

Générer : cours, 3 exercices progressifs, 1 TP noté, 1 évaluation formative. Utiliser few-shot + CoT pour chaque ressource. Itérer sur les prompts insuffisants.

3
45 min

Vérification critique

Tester le code Python dans un notebook. Vérifier la rigueur mathématique. Annoter les erreurs détectées et reformuler les prompts en conséquence.

4
45 min

Partage collectif

Présenter 1 ressource au groupe (5 min). Débat : quelle part du travail reste à l'enseignant ? Construire un template de prompt réutilisable et partagé.

Livrable attendu : Une séquence NSI complète sur un thème de Terminale, comprenant cours, exercices progressifs, TP noté et évaluation. Les prompts utilisés sont documentés. La séquence est utilisable en classe dans les 3 semaines suivantes.
