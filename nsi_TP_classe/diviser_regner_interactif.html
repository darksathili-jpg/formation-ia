Le paradigme « Diviser pour régner »

Le paradigme diviser pour régner (Divide and Conquer en anglais) est une stratégie algorithmique fondamentale qui consiste à résoudre un problème complexe en le décomposant en sous-problèmes plus simples, de même nature, jusqu'à atteindre des cas triviaux à résoudre directement.

Les 3 phases obligatoires :
1
Diviser : Découper le problème de taille n en k sous-problèmes de taille n/k
2
Régner : Résoudre chaque sous-problème récursivement (ou directement si cas de base)
3
Combiner : Fusionner les solutions des sous-problèmes pour obtenir la solution globale
⚠️ Condition d'arrêt — le cas de base :
Toute fonction récursive doit posséder un cas de base : une condition qui termine la récursion sans appel récursif. Sans cas de base, la fonction s'appellera indéfiniment et provoquera un dépassement de pile (RecursionError en Python).

Pré-condition : la liste doit être triée.

L'idée est de comparer l'élément cherché avec l'élément central de la liste. Selon le résultat, on continue uniquement sur la moitié gauche ou droite. À chaque étape, on divise l'espace de recherche par 2.

📌 Méthode :
• gauche = 0, droite = len(liste) - 1
• Tant que gauche <= droite :
  — milieu = (gauche + droite) // 2
  — Si liste[milieu] == cible → trouvé, retourner milieu
  — Si liste[milieu] < cible → chercher à droite : gauche = milieu + 1
  — Sinon → chercher à gauche : droite = milieu - 1
• Si sorti de la boucle → retourner -1 (absent)

Exemple sur [1, 3, 5, 7, 9, 11, 13] en cherchant 7 :

Étape 1 : [1, 3, 5, 7, 9, 11, 13]  milieu=3  liste[3]=7 → TROUVÉ à l'index 3 !

Exemple en cherchant 11 :
Étape 1 : [1, 3, 5, 7, 9, 11, 13]  milieu=3  liste[3]=7 < 11 → droite
Étape 2 : [9, 11, 13]              milieu=5  liste[5]=11 → TROUVÉ à l'index 5 !

Complexité temporelle : À chaque étape on divise par 2. Pour une liste de taille \(n\), on effectue au maximum \(\lfloor \log_2 n \rfloor + 1\) comparaisons.

Complexité : \(\mathcal{O}(\log n)\) dans le pire cas et en moyenne.
Comparaison : pour \(n = 1\,000\,000\), au plus 20 comparaisons car \(\log_2(10^6) \approx 20\) !
La recherche séquentielle nécessiterait jusqu'à \(1\,000\,000\) comparaisons. 
def recherche_dichotomique(liste, cible):
    """Retourne l'indice de cible dans liste (triée), ou -1 si absent."""
    gauche, droite = 0, len(liste) - 1
    while gauche <= droite:
        milieu = (gauche + droite) // 2
        if liste[milieu] == cible:
            return milieu          # trouvé
        elif liste[milieu] < cible:
            gauche = milieu + 1   # chercher à droite
        else:
            droite = milieu - 1   # chercher à gauche
    return -1                    # non trouvé

La fusion est l'étape Combiner du tri fusion. On parcourt simultanément deux listes déjà triées avec deux indices i et j, et on ajoute à chaque étape le plus petit des deux éléments courants dans la liste résultat.

📌 Algorithme de fusion :
• Initialiser resultat = [], i = 0, j = 0
• Tant que i < len(gauche) ET j < len(droite) :
  — Si gauche[i] <= droite[j] → ajouter gauche[i], incrémenter i
  — Sinon → ajouter droite[j], incrémenter j
• Ajouter les éléments restants de gauche[i:] ou droite[j:] 
def fusionner(gauche, droite):
    """Fusionne deux listes triées en une liste triée."""
    resultat = []
    i = j = 0
    while i < len(gauche) and j < len(droite):
        if gauche[i] <= droite[j]:
            resultat.append(gauche[i])
            i += 1
        else:
            resultat.append(droite[j])
            j += 1
    return resultat + gauche[i:] + droite[j:]

Le tri fusion applique le paradigme diviser pour régner au tri : on divise la liste en deux moitiés, on trie chacune récursivement, puis on les fusionne.

Arbre de récursion pour [5, 3, 8, 1, 9, 2] :

  ┌──┴──┐   │              ┌──┴──┐    │

  [5]    [3]           [1]    [9]
         ↓ fusionner ↓         ↓ fusionner ↓
   [3, 5]      [8]          [1, 9]     [2]
         ↓ fusionner ↓         ↓ fusionner ↓
      [3, 5, 8]             [1, 2, 9]
                  ↓ fusionner ↓
            [1, 2, 3, 5, 8, 9]
def tri_fusion(liste):
    """Trie une liste par la méthode du tri fusion (récursif)."""
    if len(liste) <= 1:          # cas de base : liste vide ou singleton
        return liste
    milieu = len(liste) // 2
    gauche = tri_fusion(liste[:milieu])   # régner sur la moitié gauche
    droite = tri_fusion(liste[milieu:])   # régner sur la moitié droite
    return fusionner(gauche, droite)       # combiner
Complexité du tri fusion :
• Diviser : \(\mathcal{O}(1)\) — couper en deux
• Régner : 2 appels récursifs sur des listes de taille \(n/2\)
• Combiner : \(\mathcal{O}(n)\) — parcourir les deux moitiés
Relation de récurrence : \[ T(n) = 2\,T\!\left(\frac{n}{2}\right) + \mathcal{O}(n) \] Par le théorème maître : \[ T(n) = \mathcal{O}(n \log n) \quad \text{dans tous les cas} \]
⚠️ Avantage et inconvénient :
✅ Complexité garantie \(\mathcal{O}(n \log n)\) dans tous les cas (meilleur, moyen, pire)
❌ Utilise de la mémoire supplémentaire \(\mathcal{O}(n)\) pour les listes temporaires

Calculer \(x^n\) naïvement demande \(n-1\) multiplications, soit \(\mathcal{O}(n)\). L'exponentiation rapide exploite diviser pour régner pour atteindre \(\mathcal{O}(\log n)\).

📌 Principe :
• Si \(n = 0\) → retourner \(1\) (cas de base, car \(x^0 = 1\))
• Si \(n\) est pair : \[ x^n = \left(x^{n/2}\right)^2 \] → un seul appel récursif, puis un carré
• Si \(n\) est impair : \[ x^n = x \cdot x^{n-1} = x \cdot \left(x^{(n-1)/2}\right)^2 \] 
def puissance(x, n):
    """Calcule x^n par exponentiation rapide — O(log n)."""
    if n == 0:
        return 1
    if n % 2 == 0:                 # exposant pair
        moitie = puissance(x, n // 2)
        return moitie * moitie
    else:                           # exposant impair
        moitie = puissance(x, (n-1) // 2)
        return x * moitie * moitie
Exemple : puissance(2, 10)
\[ 2^{10} \xrightarrow{\text{pair}} \left(2^5\right)^2 \xrightarrow{\text{impair}} \left(2 \cdot \left(2^2\right)^2\right)^2 \xrightarrow{\text{pair}} \cdots \xrightarrow{} 1024 \] Total : seulement 4 appels récursifs au lieu de 9 multiplications !

Complexité — Tri fusion ★★☆ Intermédiaire

Quelle est la complexité temporelle du tri fusion dans le pire cas ? Exprime ta réponse avec la notation O grand.

Compte le nombre de niveaux dans l'arbre de récursion et le travail effectué à chaque niveau.

Observe l'arbre de récursion dans le cours : combien y a-t-il de niveaux ? Et quel est le travail total effectué à chaque niveau (somme de toutes les fusions) ?
Il y a \(\log n\) niveaux dans l'arbre (on divise par 2 à chaque fois). À chaque niveau, on fusionne tous les éléments, ce qui prend en tout \(n\) opérations. Total \(= n \times \log n\).
La complexité est \(\mathcal{O}(n \log n)\). C'est optimal pour un tri par comparaison : aucun algorithme de ce type ne peut faire mieux que \(\mathcal{O}(n \log n)\) dans le pire cas.

Exponentiation rapide ★★★ Avancé

Complète la fonction puissance(x, n) qui calcule \(x^n\) par exponentiation rapide en \(\mathcal{O}(\log n)\). L'astuce : si \(n\) est pair, \(x^n = \left(x^{n/2}\right)^2\).

La méthode naïve fait \(n\) multiplications. L'exponentiation rapide n'en fait que \(\log_2 n\). Pour \(n = 1000\), c'est 1000 multiplications contre seulement 10 !

Cas de base : \(n = 0 \Rightarrow\) retourner \(1\) (car \(x^0 = 1\) pour tout \(x\)). Pour le cas pair : on calcule la moitié puissance(x, n//2) et on fait moitie * moitie (pas deux appels séparés qui doubleraient le travail !)
Pour le cas impair : \(x^n = x \cdot x^{n-1} = x \cdot \left(x^{(n-1)/2}\right)^2\). Donc moitie = puissance(x, (n-1)//2) et return x * moitie * moitie.
Les ??? : 0 | 1 | n//2 | moitie (pair) | (n-1)//2 | moitie (impair). Trace : \(2^{10} \to (2^5)^2 \to (2 \cdot (2^2)^2)^2 \to \cdots \to 1024\).
